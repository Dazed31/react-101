# Project explained — react-101

A guided, beginner-friendly explanation of every file and folder in this starter React + Vite project. If you are new to React, this file will explain what each file does, why it exists, and point out the most important lines of code so you can start editing with confidence.

Who this is for

- You are new to React and modern frontend tooling (Vite). You understand basic JavaScript but have not worked with React or build tools before.

What this document contains

- Short contract: inputs/outputs, success criteria
- An explanation for each top-level file and folder in the repo
- A line-by-line walkthrough of `src/main.jsx` and `src/App.jsx` (the most important files)
- How assets and CSS are handled
- How to run the app locally and quick edits you can try
- Next steps and small improvements you can make

Contract (tiny)

- Input: This repository (files in the workspace).
- Output: A working dev server (Vite) showing the UI in the browser and the mental model to modify it.
- Success: You can run `npm run dev`, see the app, change `src/App.jsx` and observe HMR (hot reloading). No runtime errors.

Edge cases and gotchas

- Browser caching: sometimes old assets appear; refresh or restart Vite if HMR behaves oddly.
- Node / npm versions: uses modern ecosystem (ES modules). If you see syntax errors, check Node >=14 (prefer 16+).
- Missing root element: index.html must contain the `#root` div; otherwise createRoot will fail.

Files and folders (top-level)

- `package.json`
  - Purpose: describes the project, lists dependencies and dev commands.
  - Key fields in this project:
    - `scripts`: `dev` starts the Vite dev server, `build` creates a production bundle, `preview` runs a local static server to preview the build, `lint` runs ESLint.
    - `dependencies`: `react`, `react-dom` — runtime packages required by your app.
    - `devDependencies`: tooling (Vite, ESLint, type definitions, React plugin). These are only used during development/build.
  - Why it matters: `npm install` (or `npm ci`) reads this file to install the packages your project needs.

- `package-lock.json`
  - Auto-generated by npm. It pins exact dependency versions so installs are reproducible. You usually don't edit this by hand.

- `.gitignore`
  - Lists files that Git should ignore (e.g., node_modules, build output). Not essential to run the app but important for source control hygiene.

- `index.html`
  - The single HTML page served by Vite. It contains:
    - Standard metadata (charset, viewport, title).
    - A `<div id="root"></div>` element — this is the mount point where React places your app.
    - A `<script type="module" src="/src/main.jsx"></script>` line that loads your React code as an ES module.
  - Why it matters: React manipulates the DOM inside the `#root` element. The HTML file is minimal because Vite and React favour single-page apps where JS handles rendering.

- `vite.config.js`
  - Vite's configuration. This project uses `@vitejs/plugin-react`, which enables JSX transformation and some React-specific optimizations.
  - You don't need to change it for most beginners, but it's where more advanced options (server proxy, build tweaks) would go.

- `eslint.config.js`
  - ESLint configuration file. It sets linting rules and plugins used to keep code consistent and catch mistakes.
  - The starter uses rules for JS/JSX and React hooks.

- `README.md`
  - A short README that usually explains the template and points to further documentation.

- `public/`
  - A folder of static assets that are copied as-is into the production `dist/` build and served from the root (`/`).
  - Files here include `favicon.svg` and `icons.svg` (an SVG sprite used inside the UI). Use this folder for static files that should keep their paths.

- `src/` (the most important folder)
  - This is where your app code lives. Files here are processed by Vite.
  - Contents of note in this project:
    - `main.jsx`: app entry and React bootstrapping code.
    - `App.jsx`: the main React component (the UI you see on load).
    - `index.css`: global baseline styles (fonts, colors, layout). Loaded in `main.jsx`.
    - `App.css`: component-level styles used by `App.jsx`.
    - `assets/`: images imported by the app (React/Vite allow importing assets directly from JS/JSX).


Detailed walkthrough: `src/main.jsx`

File contents (summary):

```jsx
import { StrictMode } from 'react'
import { createRoot } from 'react-dom/client'
import './index.css'
import App from './App.jsx'

createRoot(document.getElementById('root')).render(
  <StrictMode>
    <App />
  </StrictMode>,
)
```

Line-by-line explanation:

- `import { StrictMode } from 'react'`
  - `StrictMode` is a wrapper component shipped with React. It helps highlight potential problems in an app during development (like deprecated APIs, unexpected side-effects). It has no effect in production builds.
- `import { createRoot } from 'react-dom/client'`
  - Modern React uses `createRoot` (introduced in React 18) to create the root that manages the render. This is where concurrent features come from.
- `import './index.css'`
  - Imports global CSS. Vite processes this and injects it into the document. This file sets color variables, base font sizes, and layout rules for the whole app.
- `import App from './App.jsx'`
  - Imports the default export from `App.jsx`, which is the top-level React component for this app.
- `createRoot(document.getElementById('root')).render(...)`
  - Finds the DOM element with id `root` (from `index.html`) and tells React to render the element tree into that DOM node.
  - Inside `render`, the app is wrapped with `StrictMode` and the `App` component is rendered.

Why this file exists

- The file wires React to the real DOM (document). It's minimal by design: the app logic should live in components, not here.


Detailed walkthrough: `src/App.jsx`

This is the starter UI. It's a React component defined as a JavaScript function and exported as the default export.

Key concepts used in this file

- JSX: a syntax that looks like HTML inside JavaScript. It's how you declare UI in React.
- Components: `App` is a component — a function that returns JSX describing what the UI should look like.
- Hooks: `useState` is a built-in hook that lets a function component hold state.
- Asset imports: images are imported (e.g., `import reactLogo from './assets/react.svg'`) and used as `src` in `<img />` tags. Vite transforms these imports into URLs.

Important lines and explanation

- `import { useState } from 'react'`
  - `useState` is a hook used to store and update local component state.

- `const [count, setCount] = useState(0)`
  - Creates a state variable named `count` with initial value `0` and a function `setCount` to update it.
  - When `setCount` is called, React re-renders the component and the UI updates to show the new value.

- The returned JSX contains:
  - A `section` with id `center` that groups the hero (images) and a button.
  - An interactive button:

```jsx
<button onClick={() => setCount((count) => count + 1)}>Count is {count}</button>
```

When clicked, this uses `setCount` to increment the counter. The displayed text updates automatically.

- `<img src={heroImg} ... />` and other images
  - Because images are imported, the bundler resolves them to correct URLs.

Why function components and hooks

- Function components are the modern, recommended way to write React components.
- Hooks like `useState` let function components keep state without switching to class components.


Styling and assets

- CSS files:
  - `src/index.css` is included in `main.jsx` and provides global variables and page layout styles.
  - `src/App.css` is imported by `App.jsx` and contains styles specific to the elements rendered by `App` (hero, logos, button, layout tweaks).
  - CSS in this project uses standard selectors and nested style hints (the file content contains nested selectors in the SCSS-like style but it is plain CSS with some `@media` and pseudoselectors; Vite processes it normally).

- assets folder:
  - `src/assets/react.svg`, `vite.svg`, `hero.png` are image files used by `App.jsx`.
  - In JS/JSX you can import them directly (e.g., `import reactLogo from './assets/react.svg'`) — Vite turns that into a URL string at build time.


Public folder

- `public/favicon.svg` — browser favicon referenced from `index.html`.
- `public/icons.svg` — an SVG sprite containing multiple symbols used with `<use href="/icons.svg#symbol-id">`. These are static and referenced by absolute paths (`/icons.svg`) because everything in `public/` is served from the root.


ESLint config

- `eslint.config.js` sets up lint rules for JavaScript and React hooks. It helps catch errors and enforce a consistent code style. You can run `npm run lint` to inspect issues.


How to run the project (quick)

1. Install dependencies (one-time):

```bash
npm install
```

1. Start the dev server with Vite:

```bash
npm run dev
```

1. Open your browser at the address Vite prints (usually <http://localhost:5173>).

What to try first (interactive learning)

- Open `src/App.jsx` and change the text inside the `<h1>` (e.g., to your name). Save — the browser should update instantly (HMR).
- Change the initial count in `useState(0)` to another number and observe the starting value.
- Remove `StrictMode` in `main.jsx` and notice nothing in the UI changes — but keep it for development safety.


Small, safe improvements you can make now

- Add a new component file `src/Hello.jsx` and import it in `App.jsx`. Keep it simple:

```jsx
function Hello({ name }) {
  return <div>Hello {name}</div>
}

export default Hello
```

- Add an `alt` text to decorative images (some are empty strings in the starter); accessibility matters.
- Add TypeScript later (rename files to `.tsx`) if you want typed components — the repo already includes `@types/react`.


Notes about build and production

- `npm run build` creates an optimized `dist/` folder ready to be deployed.
- `npm run preview` will start a local static server that serves the production build so you can test it locally.

Where to go next (learning path)

1. Learn JSX: treat it like JavaScript expressions that return UI trees.
2. Learn components: split parts of `App.jsx` into smaller components (Hero, Docs, Social).
3. Learn props: pass data into components to make them reusable.
4. Learn effects: `useEffect` for side effects (fetching data, subscriptions).
5. Try routing (React Router) and state management if your app grows.
